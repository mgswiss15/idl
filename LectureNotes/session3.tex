\section{Deep Neural Networks \& Training}
\slideref{Session 3}

These notes accompany the third lecture. In Session~2 we established that a one-hidden-layer network with ReLU activations can approximate any continuous function - but we stopped short of asking how to actually \emph{find} good parameters. This session addresses that question. We first generalise the architecture to arbitrary depth, then introduce the machinery needed to train it: a loss function to measure error, and gradient descent to minimise it.

% - - - - - - - - - - - - - - - - - --
\subsection{From Shallow to Deep Networks}
\slideref{Session 3: From Shallow to Deep, Matrix Notation}

The one-hidden-layer network from Session~2 computes, for input $\vx \in \R^d$:
\begin{align}
h_j &= \act\!\left(\theta_{j0} + \sum_{i=1}^{d} \theta_{ji}\, x_i\right), \qquad j = 1, \ldots, k \\
\yhat &= \theta_0 + \sum_{j=1}^{k} \theta_j\, h_j.
\end{align}
The natural extension is to stack additional layers: the output of one layer becomes the input of the next. Each layer applies a linear transformation followed by a nonlinear activation. Stacking layers allows the network to learn increasingly abstract representations - the first layer might detect simple patterns in the input, the second layer combinations of those patterns, and so on. Figure~\ref{fig:shallow_vs_deep} contrasts the shallow network from Session~2 with a two-hidden-layer deep network.

\begin{figure}[h]
    \centering
    \begin{minipage}{0.45\textwidth}
        \centering
        \resizebox{\textwidth}{!}{\input{\CommonPath/tikz/mlp_multi_input_y.tex}}
    \end{minipage}
    \hfill
    \begin{minipage}{0.50\textwidth}
        \centering
        \resizebox{\textwidth}{!}{\input{\CommonPath/tikz/mlp_deep.tex}}
    \end{minipage}
    \caption{Left: shallow network with one hidden layer. Right: deep network with two hidden layers. Each additional layer introduces another linear transform followed by a nonlinearity.}
    \label{fig:shallow_vs_deep}
\end{figure}

To handle multiple layers cleanly, scalar notation becomes cumbersome. We start by collecting the individual quantities into vectors. The inputs $x_1, \ldots, x_d$ are gathered into an input vector $\vx \in \R^d$, and the hidden neuron activations $h_1, \ldots, h_k$ into a hidden activation vector $\vh \in \R^k$. Correspondingly, the weights and biases are collected into a matrix and a vector. The bias of each output neuron $j$ gives a bias vector $\pvec_0 \in \R^k$, and the weights connecting all $d$ input neurons to all $k$ output neurons form a weight matrix $\boldsymbol{\Theta} \in \R^{k \times d}$:
\begin{equation}
\boldsymbol{\Theta} =
\begin{pmatrix}
\theta_{11} & \cdots & \theta_{1d} \\
\vdots & \ddots & \vdots \\
\theta_{k1} & \cdots & \theta_{kd}
\end{pmatrix}
\in \R^{k \times d}, \qquad
\pvec_0 =
\begin{pmatrix} \theta_{10} \\ \vdots \\ \theta_{k0} \end{pmatrix}
\in \R^k.
\end{equation}
The matrix $\boldsymbol{\Theta}$ has $k$ rows and $d$ columns. Each row $j$ contains the weights $(\theta_{j1}, \ldots, \theta_{jd})$ connecting all $d$ input neurons to output neuron $j$ - so there is one row per output neuron. Each column $i$ contains the weights $(\theta_{1i}, \ldots, \theta_{ki})$ connecting input neuron $x_i$ to all $k$ output neurons - so there is one column per input neuron. The bias vector $\pvec_0 \in \R^k$ has one entry per output neuron.

\begin{wrapfigure}{r}{0.35\textwidth}
\centering
\resizebox{0.33\textwidth}{!}{\input{\CommonPath/tikz/matrix_vector_multiply.tex}}
\caption{Hidden layer computation as matrix-vector operation.}
\label{fig:matrix_vector_multiply}
\end{wrapfigure}

With this notation, the entire hidden layer computation collapses to a single vector equation:
\begin{equation}
\vh = \act\!\left(\pvec_0 + \boldsymbol{\Theta}\vx\right) \in \R^k,
\label{eq:layer_matrix}
\end{equation}
where $\act$ is applied elementwise. The matrix-vector product $\boldsymbol{\Theta}\vx \in \R^{k \times d} \cdot \R^d = \R^k$ produces one pre-activation value per output neuron. Adding the bias $\pvec_0 \in \R^k$ and applying $\act$ elementwise gives the hidden activation vector $\vh \in \R^k$.

\subsection{Layer Notation and Function Composition}
\slideref{Session 3: Layer Notation, Deep Network as Function Composition, Dimensions}

We index layers $l = 0, 1, \ldots, L$, where $l=0$ denotes the input and $l=L$ the output. The forward pass is defined recursively:
\begin{equation}
\vh^{(0)} = \vx, \qquad \vh^{(l)} = \act\!\left(\thetavec{l}_0 + \thetamat{l}\, \vh^{(l-1)}\right), \quad l = 1, \ldots, L-1,
\end{equation}
with output
\begin{equation}
\yhat = \thetavec{L}_0 + \thetamat{L}\, \vh^{(L-1)}.
\end{equation}



\begin{wrapfigure}{r}{0.5\textwidth}
\centering
\resizebox{0.48\textwidth}{!}{\input{\CommonPath/tikz/mlp_layer_notation.tex}}
\caption{Layer notation.}
\label{fig:layer_notation}
\end{wrapfigure}
The output layer applies no activation function - the raw linear output $\yhat$ is passed directly as the network's output. This is the general convention for all network types: for regression, $\yhat$ is used directly as the prediction; for classification, $\yhat$ contains the raw logits to which a task-specific transformation (sigmoid, softmax) is applied afterwards, outside the network. The layer structure is illustrated in Figure~\ref{fig:layer_notation}.

At layer $l$, the weight matrix has shape $\thetamat{l} \in \R^{k_l \times k_{l-1}}$ and the bias vector $\thetavec{l}_0 \in \R^{k_l}$, with $k_0 = d$. The matrix-vector product $\thetamat{l}\vh^{(l-1)} \in \R^{k_l \times k_{l-1}} \cdot \R^{k_{l-1}} = \R^{k_l}$ produces one pre-activation value per output neuron in layer $l$.

Viewed as a whole, the deep network is a composition of functions:
\begin{equation}
\yhat = f_\theta(\vx) = \left(h^{(L)}_\theta \circ \cdots \circ h^{(1)}_\theta\right)(\vx) = \thetavec{L}_0 + \thetamat{L}\,\act\!\left(\cdots\,\act\!\left(\thetavec{1}_0 + \thetamat{1}\vx\right)\right).
\end{equation}

The total number of parameters is the sum over all layers of the number of weights plus biases:
\begin{equation}
|\pvec| = \sum_{l=1}^{L} k_l \cdot (k_{l-1} + 1).
\end{equation}

\begin{wrapfigure}{r}{0.52\textwidth}
\centering
\resizebox{0.50\textwidth}{!}{\input{\CommonPath/tikz/mlp_dimensions.tex}}
\caption{Network with $d=3$, $k_1=k_2=4$, $m=2$. Total: $16+20+10=46$ parameters.}
\label{fig:mlp_dimensions}
\end{wrapfigure}

For example, the small network in Figure~\ref{fig:mlp_dimensions} with $d=3$ inputs, two hidden layers of width $k_1 = k_2 = 4$, and $m=2$ outputs has:
\begin{align}
& 4 \cdot (3+1) + 4 \cdot (4+1) + 2 \cdot (4+1) \nonumber\\
&= 16 + 20 + 10 = 46 \text{ parameters.}
\end{align}
This is a toy example. In practice, networks are much wider and deeper - parameter counts grow quadratically with layer width and linearly with depth. A fully-connected network with $d=784$ inputs (a $28\times28$ image), two hidden layers of width $1024$, and $10$ outputs already has approximately $1.8$ million parameters. Modern architectures are far larger: ResNet-50, a standard image classification model, has 25 million parameters; GPT-2 has 1.5 billion; GPT-3 has 175 billion. Finding good values for all parameters jointly from data is the central challenge of deep learning.

\subsection{Batch Formulation}
\slideref{Session 3: Batch Formulation}

In practice, we process multiple training samples simultaneously in a \textbf{batch}. Given $N$ samples, we stack the inputs as rows of a matrix $\mathbf{X} \in \R^{N \times d}$, and the hidden activations of layer $l$ form a matrix $\mathbf{H}^{(l)} \in \R^{N \times k_l}$.

\begin{wrapfigure}{l}{0.4\textwidth}
\centering
\resizebox{0.38\textwidth}{!}{\input{\CommonPath/tikz/batch_matrix_multiply.tex}}
\caption{Batch forward pass.}
\label{fig:batch_matrix_multiply}
\end{wrapfigure}

The batch forward pass for layer $l$ is:
\begin{equation}
\mathbf{H}^{(l)} = \act\!\left(\mathbf{1}\thetavec{l}_0{}^\top + \mathbf{H}^{(l-1)} \left(\thetamat{l}\right)^\top\right),
\end{equation}
where $\mathbf{1} \in \R^N$ broadcasts the bias across all samples, as illustrated in Figure~\ref{fig:batch_matrix_multiply}. This is precisely what PyTorch computes with \texttt{H = X @ W.T + b}: the batch dimension comes first, and the matrix multiply operates across the feature dimension.

Processing data in batches is computationally efficient: modern hardware (GPUs, TPUs) is designed for large parallel matrix operations, and a single batched matrix multiply is far faster than $N$ sequential vector operations.

\subsection{The Learning Problem}
\slideref{Session 3: The Learning Problem}

We now have a network $f_\theta$ and a dataset $\dataset = \{(\vx^{(i)}, y^{(i)})\}_{i=1}^{\nsamples}$. The goal is to find parameters $\pvec$ such that $f_\theta(\vx) \approx y$ on unseen data. This raises two immediate questions:

\begin{enumerate}
    \item How do we \textbf{measure} how wrong $f_\theta$ is? $\Rightarrow$ \emph{loss function}
    \item How do we \textbf{find} good $\pvec$ systematically? $\Rightarrow$ \emph{optimisation}
\end{enumerate}

\begin{wrapfigure}{r}{0.35\textwidth}
\centering
\resizebox{0.33\textwidth}{!}{\input{\CommonPath/tikz/learning_problem.tex}}
\caption{Many functions fit the data. Which is best?}
\label{fig:learning_problem}
\end{wrapfigure}

As illustrated in Figure~\ref{fig:learning_problem}, there are infinitely many functions that could plausibly fit the training data - they differ in how they interpolate between observed points and how they extrapolate beyond them. With a low-dimensional input we could perhaps inspect candidate functions visually. But $f_\theta$ maps from $\R^d$ where $d$ may be hundreds or thousands - visual inspection is impossible. Trial and error is equally hopeless: the parameter space has millions of dimensions, and randomly trying parameter configurations would never converge to anything useful.

What we need is a single scalar number that summarises how wrong the current $f_\theta$ is on the training data - small when the predictions are good, large when they are bad. This scalar is the \textbf{loss}, and it turns the vague goal of ``find a good function'' into a concrete mathematical optimisation problem: minimise the loss over all possible parameter settings.

\subsection{Loss Functions}
\slideref{Session 3: What is a Loss Function?, Regression: Mean Squared Error}

A \textbf{loss function} $\ell(\yhat, y)$ measures the discrepancy between a prediction $\yhat$ and a true label $y$ for a single sample. It satisfies $\ell(\yhat, y) = 0$ when $\yhat = y$ and $\ell(\yhat, y) > 0$ otherwise, growing larger as the prediction worsens. The loss is the compass that guides training: it converts the vague notion of ``a good model'' into a concrete scalar that can be minimised.

For a full dataset, we average the per-sample loss over all training examples to get the \textbf{dataset loss}:
\begin{equation}
\loss(\pvec) = \frac{1}{\nsamples} \sum_{i=1}^{\nsamples} \ell\!\left(f_\theta(\vx^{(i)}), y^{(i)}\right).
\end{equation}
Averaging rather than summing makes $\loss$ independent of dataset size, which is convenient when comparing runs with different amounts of data or different batch sizes.

The choice of $\ell$ depends on the task. For regression we use Mean Squared Error; for binary classification, Binary Cross-Entropy; for multiclass classification, Cross-Entropy with a softmax output. Their probabilistic motivation via maximum likelihood estimation is developed in a later session. Here we focus on MSE for regression; classification losses are covered when we discuss classification in detail.

\subsubsection{Mean Squared Error}

\begin{wrapfigure}{r}{0.35\textwidth}
\centering
\resizebox{0.33\textwidth}{!}{\input{\CommonPath/tikz/mse_loss.tex}}
\caption{MSE loss. Vertical lines show residuals $\yhat^{(i)} - y^{(i)}$.}
\label{fig:mse_loss}
\end{wrapfigure}

For regression with scalar output $y \in \R$ and prediction $\yhat \in \R$, the per-sample loss is the squared residual:
\begin{equation}
\ell(\yhat, y) = (\yhat - y)^2.
\end{equation}
The dataset loss is accordingly:
\begin{equation}
\loss(\pvec) = \frac{1}{\nsamples}\sum_{i=1}^{\nsamples}\!\left(\yhat^{(i)} - y^{(i)}\right)^2.
\end{equation}
As shown in Figure~\ref{fig:mse_loss}, the residuals $\yhat^{(i)} - y^{(i)}$ are the vertical distances between the predicted function and the data points. MSE averages the squares of these distances. Three properties make squaring the natural choice. First, prediction errors can be positive or negative and would cancel if averaged directly - squaring removes the sign. Second, squaring penalises large errors more heavily than small ones: a prediction off by 10 contributes 100 times more to the loss than one off by 1. Third, the squared function is differentiable everywhere, which is essential for gradient-based optimisation.

\subsection{Gradient Descent}
\slideref{Session 3: The Optimization Problem, Gradient Descent, The Key Question}

Training the network means finding parameters that minimise the dataset loss:
\begin{equation}
\pvec^* = \arg\min_{\pvec}\ \loss(\pvec).
\end{equation}
Unlike linear regression, no closed-form solution exists: $\loss(\pvec)$ is highly nonlinear in $\pvec$ through the nested compositions of weight matrices and activations, the parameter space has millions of dimensions, and the loss surface is non-convex with many local minima and saddle points. We need an iterative strategy that navigates this landscape using only local information.

The key local information at any point $\pvec$ is the \textbf{gradient} $\nabla_{\pvec} \loss(\pvec)$ - the vector of partial derivatives of $\loss$ with respect to each parameter:
\begin{equation}
\nabla_{\pvec} \loss(\pvec) = \frac{\partial \loss}{\partial \pvec} = \left(\frac{\partial \loss}{\partial \theta_1}, \frac{\partial \loss}{\partial \theta_2}, \ldots\right).
\end{equation}
The gradient points in the direction of steepest \emph{ascent} of the loss surface. To \emph{minimise} the loss we step in the opposite direction, giving the \textbf{gradient descent} update rule:
\begin{equation}
\pvec \leftarrow \pvec - \eta\, \nabla_{\pvec} \loss(\pvec),
\end{equation}
where $\eta > 0$ is the \textbf{learning rate}, controlling the step size. Starting from a random initialisation, we repeatedly apply this update until the loss stops improving.

\begin{wrapfigure}{r}{0.42\textwidth}
\centering
\resizebox{0.40\textwidth}{!}{\input{\CommonPath/tikz/gradient_descent_2d.tex}}
\caption{Gradient descent on a quadratic loss. Steps from $\theta_0$ towards minimum $\theta^*$.}
\label{fig:gradient_descent}
\end{wrapfigure}

Figure~\ref{fig:gradient_descent} illustrates gradient descent on a one-dimensional quadratic loss $\loss(\theta) = \theta^2$. Starting from $\theta_0 > 0$, the gradient $\frac{\partial \loss}{\partial \theta} = 2\theta$ is positive, so the update $\theta \leftarrow \theta - \eta \cdot 2\theta$ moves $\theta$ to the left, towards the minimum at $\theta^* = 0$. As $\theta$ decreases, the gradient decreases too, so subsequent steps are naturally smaller even with a fixed learning rate — the algorithm automatically slows down as it approaches the optimum.

The learning rate $\eta$ is a critical hyperparameter. Too large and the updates overshoot the minimum, bouncing back and forth or even diverging. Too small and convergence is impractically slow. In practice, $\eta$ is carefully tuned and often adapted during training - a topic we return to in the dedicated optimisation session.

The central computational challenge is evaluating $\nabla_{\pvec} \loss(\pvec)$ efficiently. The loss has millions of components in $\pvec$, and computing each partial derivative $\frac{\partial \loss}{\partial \theta_j}$ separately would require one forward pass per parameter - completely intractable. The answer is \textbf{backpropagation}: an algorithm that computes all partial derivatives simultaneously in a single backward pass, at a cost comparable to a single forward pass. We develop it in the next session.
